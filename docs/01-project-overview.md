# 📋 プロジェクト概要・要件定義書

## AsahiRoute - 新聞配達ルート最適化アプリ

---

## 1. プロジェクト背景

### 1.1 現状の課題

| 課題 | 詳細 |
|------|------|
| **順路帳の紙ベース管理** | 配達順序が紙の順路帳に依存しており、新人・高齢者・外国人留学生にとって習得が困難 |
| **留守止め対応の遅延** | 購読者の一時停止情報が口頭・手書きで伝達され、配達ミスが発生 |
| **新規購読者の挿入** | 新しい配達先の追加時にルート全体の見直しが必要 |
| **言語バリア** | 外国人留学生が日本語の順路帳を読めず、業務習得に時間がかかる |
| **高齢配達員の負担** | 小さい文字・複雑な地図の読み取りが困難 |

### 1.2 解決目標

- 順路帳をデジタル化し、**地図ベースのビジュアルナビゲーション**を提供
- **多言語対応**で外国人留学生の即戦力化
- **大きな文字・音声案内**で高齢者をサポート
- **リアルタイム更新**で留守止め・新規挿入を即時反映
- **管理者向けダッシュボード**で効率的な区域管理

## 2. ターゲットユーザー

### 2.1 配達員側（デリバリーアプリ）

| ユーザー層 | ニーズ | 優先機能 |
|-----------|--------|---------|
| **新人配達員** | ルートの素早い習得 | ターンバイターンナビ、ビジュアルマップ |
| **高齢配達員** | 視認性・操作のしやすさ | 大文字UI、音声案内、シンプル操作 |
| **外国人留学生** | 言語サポート | 多言語UI（越・中・英・韓・ネパール語等） |
| **ベテラン配達員** | 変更情報の即時把握 | 留守止め/新規通知、差分表示 |

### 2.2 管理者側（管理ダッシュボード）

| ユーザー層 | ニーズ | 優先機能 |
|-----------|--------|---------|
| **店長・所長** | 全体の配達状況把握 | ダッシュボード、KPI表示 |
| **区域管理者** | 担当区域の管理 | 区域編集、配達員割当 |
| **事務員** | 購読者情報管理 | 購読者CRUD、留守止め管理 |

## 3. 機能要件

### 3.1 配達員アプリ（モバイル - PWA/ネイティブ）

#### 🗺️ F1: デジタル順路帳（ルートナビゲーション）
- 配達先を順番に地図上に表示
- ターンバイターンナビゲーション
- 現在位置のリアルタイム追跡
- 次の配達先までの矢印表示
- 配達先の建物写真表示（オプション）

#### 📢 F2: 留守止め表示
- 留守止め中の配達先をグレーアウト/スキップ表示
- 留守止めの期間表示（開始日〜終了日）
- 自動ルート調整（留守止め先をスキップ）
- プッシュ通知で留守止め変更を即時通知

#### ➕ F3: 新規挿入対応
- 新規購読者の配達先をハイライト表示
- 最適な挿入位置の自動提案
- 新規追加の通知
- 配達先詳細（新聞種類、部数、ポスト位置等）

#### ✅ F4: 配達完了記録
- ワンタップで配達完了マーク
- 配達できなかった場合の理由記録
- 配達時刻の自動記録
- GPS位置での配達証跡

#### 🌍 F5: 多言語対応
- 対応言語: 日本語、英語、ベトナム語、中国語、韓国語、ネパール語
- UI全体の言語切替
- 住所・備考の翻訳補助
- 音声案内の多言語対応

#### 🔊 F6: アクセシビリティ
- フォントサイズ調整（小・中・大・特大）
- ハイコントラストモード
- 音声案内（次の配達先を読み上げ）
- 夜間モード（早朝配達対応）
- ボタンサイズの拡大オプション

#### 📱 F7: オフライン対応
- ルートデータのオフラインキャッシュ
- オフラインでの配達完了記録
- オンライン復帰時の自動同期

#### 🎓 F8: オンボーディングチュートリアル（v2.0追加）
- 初回ログイン時の3ステップインタラクティブガイド
- ルート確認→配達記録→変更チェックの流れを視覚的に案内
- スキップ可能、設定から再表示可能

#### 🏃 F9: ルート学習（練習）モード（v2.0追加）
- 実配達なしでルートを仮想的に辿れるモード
- 新人研修で活用、配達記録には反映しない
- 地図/リスト両対応

#### 👆 F10: スワイプジェスチャー操作（v2.0追加）
- 右スワイプ＝配達完了、左スワイプ＝スキップ
- 閾値フィードバック（振動＋色変化）
- オンボーディングで操作方法を案内

#### 🆘 F11: 緊急SOSボタン（v2.0追加）
- 早朝・深夜配達中の緊急連絡機能
- 3秒長押しで店舗にGPS位置付きアラート送信
- 110番・119番へのワンタップ発信
- 全画面からアクセス可能（フローティングボタン）

#### 📊 F12: 配達完了サマリー（v2.0追加）
- 配達終了後の成績・所要時間・距離の要約画面
- 前回比較で改善度を表示
- モチベーション維持に寄与

#### 🎤 F13: 音声入力メモ（v2.0追加）
- 配達メモを音声で入力（Web Speech API）
- バイク・自転車運転中のハンズフリー対応

#### 🏪 F14: ランドマーク表示（v2.0追加）
- コンビニ・信号・公園等の目印を地図上に表示
- 新人・留学生の方向認識を支援

#### 🎨 F15: 色覚バリアフリーモード（v2.0追加）
- 色＋形状＋パターン＋テキストの4要素でステータス表現
- WCAG 2.1 AA準拠

#### ⛅ F16: 天候情報バナー（v2.0追加）
- 雨天・台風時の注意バナー表示
- ビニール包装確認リマインダー

#### 🔄 F17: クイック引き継ぎ（v2.0追加）
- 別配達員にルートを即時転送（代走対応）

### 3.2 管理者ダッシュボード（Web）

#### 🏘️ M1: 区域管理
- 地図上で配達区域を設定・編集
- 区域の分割・統合
- 区域ごとの購読者一覧
- 区域ごとの統計情報

#### 👥 M2: 配達員管理
- 配達員の登録・編集・削除
- 担当区域の割り当て
- シフト管理
- 配達実績の確認

#### 📰 M3: 購読者管理
- 購読者のCRUD操作
- 新聞種類・部数の管理
- 配達先住所のジオコーディング
- 一括インポート（CSV/Excel）

#### 🚫 M4: 留守止め管理
- 留守止めの登録・解除
- 期間指定（開始日〜終了日）
- 一括留守止め設定
- 留守止め履歴

#### ➕ M5: 新規挿入管理
- 新規購読者の追加
- 配達順序の自動最適化
- ルートへの挿入位置の提案
- 既存ルートへの影響確認

#### 📊 M6: ルート最適化
- AIベースのルート最適化エンジン
- 配達順序の自動計算（TSP - 巡回セールスマン問題）
- 道路条件（一方通行、階段等）の考慮
- 手動でのルート調整機能

#### 📈 M7: レポート・分析
- 日次/週次/月次の配達レポート
- 配達時間の分析
- 遅延・未配達の分析
- 配達効率のKPI表示

#### 🔔 M8: 通知管理
- 配達員への一斉通知
- 個別通知
- 留守止め/新規挿入の自動通知
- 緊急連絡機能

#### 📅 M9: カレンダービュー（留守止め）（v2.0追加）
- 月間カレンダーで留守止めを視覚的に管理
- 期間バーで重複・空き確認
- ドラッグで期間変更可能

#### 🖨️ M10: 印刷用順路帳（v2.0追加）
- バックアップ用の紙の順路帳をPDF出力
- 留守止め・新規を反映した最新版
- A4横レイアウト最適化

#### 📋 M11: 操作ログ・監査証跡（v2.0追加）
- 誰がいつ何を変更したかの履歴
- フィルタリング・検索対応
- CSV出力可能

#### 🔍 M12: グローバル検索（v2.0追加）
- 購読者名・住所・顧客コードで全体横断検索
- インクリメンタルサーチ対応

#### 📅 M13: シフトカレンダー（v2.0追加）
- 配達員のシフトをカレンダーで管理
- ドラッグ&ドロップ操作

#### 📸 M14: 一括写真アップロード（v2.0追加）
- 建物・ポスト写真のCSV＋ZIP一括登録
- 既存購読者への紐付け

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| **パフォーマンス** | アプリ起動3秒以内、ルート表示2秒以内 |
| **可用性** | 99.5%以上のアップタイム |
| **セキュリティ** | HTTPS通信、Sanctumトークン認証、個人情報暗号化、監査ログ |
| **スケーラビリティ** | 同時100店舗、10,000配達員対応 |
| **対応端末** | iOS 14+, Android 10+, Chrome/Edge/Safari最新版 |
| **オフライン** | 4時間以上のオフライン動作保証 |
| **データ保持** | 配達履歴3年間保持 |
| **バックアップ** | 日次自動バックアップ |
