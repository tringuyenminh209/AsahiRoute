## 1. モバイルアプリ設計方針

### 1.1 ターゲットデバイス・環境

```
画面サイズ: 320px〜640px（モバイルオンリー）
基準フレーム: 375×812px（iPhone 13/14基準）
向き: 縦固定（横回転無効）
動作環境: PWA（Service Worker＋IndexedDB）
使用時間帯: 早朝3:00〜6:00（暗所・低温）
片手操作: 右手親指で全主要操作が可能なゾーン設計
```

### 1.2 対象ユーザープロファイル

| ユーザー | 特性 | UI最適化 |
|---------|------|----------|
| 新人配達員 | ルート未記憶 | 地図ナビ重視、学習モード |
| 高齢者配達員 | 視力低下・操作不慣れ | 大文字・大ボタン・音声案内 |
| 外国人留学生 | 日本語不慣れ | 多言語・写真・アイコン重視 |
| ベテラン配達員 | 効率重視 | リストビュー・最小操作 |

### 1.3 モバイル専用レイアウト構造

```
┌─────────────────────┐
│    ヘッダー (48px)     │
├─────────────────────┤
│                     │
│    コンテンツ領域      │
│    (スクロール可)      │
│                     │
├─────────────────────┤
│  ボトムナビ (64px)    │
└─────────────────────┘
  [SOS] ← フローティング（右下、ナビ上32px）
```

## 4. インタラクション・マイクロインタラクション

### 4.1 配達完了フロー

```
[配達完了ボタンタップ]
  → ボタン緑遷移（150ms ease-out）
  → ✓アニメーション（弾む）
  → バイブレーション（50ms）
  → 音声: "完了しました"（ON時）
  → カードが上にスライドアウト（250ms）
  → 次の配達先が下からスライドイン（250ms）
  → 進捗バーがスムーズ増加
  → 地図マーカー青→緑
```

### 4.2 スワイプジェスチャー

```
右スワイプ（60px以上）: 配達完了
  → 背景緑スライド + ✓アイコン → 閾値超えで確定

左スワイプ（60px以上）: スキップ
  → 背景グレースライド + ⏭️アイコン → 閾値超えで確定

フィードバック: 距離に応じた透過度変化 + 閾値到達時にバイブ
```

### 4.3 リアルタイム反映

```
[WebSocket: SuspensionCreatedイベント]
  → ホーム: 変更セクションにアニメーション追加
  → ルートリスト: 対象行がグレーにフェード
  → ルートマップ: マーカー青→グレー
  → トースト: "鈴木様が留守止めになりました（4/1〜4/10）"
  → ベルバッジ+1
```

### 4.4 オフライン状態表示

```
[オフライン検知]
  → ヘッダー下に固定バナー: "📴 オフライン — データは自動同期されます"
    背景#FEF3C7、text-sm、#92400E
  → 操作は継続可能（IndexedDBに保存）
  → 同期不可アクション（SOS等）はグレーアウト＋ツールチップ

[オンライン復帰]
  → バナー: "✅ オンライン復帰 — 同期中..."（背景#DCFCE7）
  → 同期完了後バナー消滅（3秒後フェードアウト）
```

### 4.5 夜間モード自動切替

```
[時刻 3:00到達 or アプリ起動時3:00〜6:00]
  → ダークテーマにスムーズ切替（350ms）
  → 地図タイル: CartoDB Dark Matterに変更
  → ステータスカラー: ダーク用明度調整版に切替
  → 輝度自動低減（目の負担軽減）

[時刻 6:00到達]
  → ライトテーマに復帰（350ms）
```
